My situation:
I am using firebase/firestore, resend and google calendar. Please give me a step by step detailed instruction on how to do that? I already have nodemail to send me "Contact Us" emails directly and using resend to send the confirmation email after they book. I am not on the blaze plan

Suggested plan:
For your exact setup, the simplest path is:
Use Resend’s scheduled emails for the reminders, and store the reminder IDs in Firestore.
Because you’re not on Firebase Blaze, I would not build this with Firebase scheduled functions right now. Firebase’s own docs say deploying Cloud Functions requires the pay-as-you-go Blaze plan, and scheduled functions use Cloud Scheduler, which is billed.

This works well because Resend can:

schedule emails for a specific time,
reschedule them,
cancel them,
and schedule up to 30 days ahead.
Also, scheduled emails cannot be sent via SMTP, so this should use the Resend API/SDK, not Nodemailer SMTP.
What you should build

When someone books:

Save the booking in Firestore
Create the Google Calendar event
Send the normal confirmation email with Resend
Schedule:
one reminder 24 hours before
one reminder 2 hours before
Save the returned Resend email IDs in Firestore

Then, if they reschedule or cancel:

update or cancel the Google Calendar event
cancel the old scheduled Resend reminder emails
create new ones if needed
update Firestore
Important limitation before you start

Resend scheduled emails can only be scheduled up to 30 days in advance. So if someone books an appointment 2 months from now, you cannot schedule the 24-hour reminder immediately on booking day.

So use this rule:

if the reminder time is within 30 days, schedule it now
if it is more than 30 days away, store it in Firestore as pendingScheduleLater

For a garage site, that may be totally fine, because many bookings are within a few days or weeks anyway.

Why this matters

You need to store the Resend reminder email IDs because Resend’s update/cancel actions work on the email ID.

Step 2: Always store the appointment time in UTC

This is really important.

Save:

the appointment time as a real UTC date/timestamp
the display timezone separately

Example:

customer books April 20, 2026 at 2:00 PM Montreal time
convert that to a real Date
save it in Firestore as a Timestamp
also save "America/Montreal"

That helps prevent reminder bugs around timezone and daylight savings.

Step 3: Make a helper that calculates reminder times

Server-side:

function getReminderDates(appointmentDate) {
  const twentyFourHoursBefore = new Date(appointmentDate.getTime() - 24 * 60 * 60 * 1000);
  const twoHoursBefore = new Date(appointmentDate.getTime() - 2 * 60 * 60 * 1000);

  return {
    twentyFourHoursBefore,
    twoHoursBefore,
  };
}
Step 4: Create a helper that checks if a reminder can be scheduled now

Because of the 30-day Resend limit:

function canScheduleWithResend(targetDate) {
  const now = new Date();
  const maxDate = new Date(now.getTime() + 30 * 24 * 60 * 60 * 1000);

  return targetDate > now && targetDate <= maxDate;
}

Resend supports scheduling up to 30 days ahead.

Step 5: Create a Resend helper for scheduled reminders

Use the Resend API/SDK, not SMTP, because scheduled sending is not supported through SMTP.

Example:

import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY);

async function scheduleReminderEmail({
  to,
  subject,
  html,
  scheduledAt,
}) {
  const { data, error } = await resend.emails.send({
    from: "Jerry Service Garage <appointments@yourdomain.com>",
    to: [to],
    subject,
    html,
    scheduledAt: scheduledAt.toISOString(),
  });

  if (error) {
    throw error;
  }

  return data?.id || null;
}

Resend’s docs show scheduledAt can be set with an ISO date, and scheduled emails can later be updated or cancelled.

Step 7: Example full booking flow skeleton
import admin from "firebase-admin";
import { google } from "googleapis";
import { Resend } from "resend";

const db = admin.firestore();
const resend = new Resend(process.env.RESEND_API_KEY);

async function createBooking(req, res) {
  try {
    const {
      customerName,
      customerEmail,
      customerPhone,
      service,
      appointmentAt,
      garageName,
      garageAddress,
    } = req.body;

    const appointmentDate = new Date(appointmentAt);

    // 1) Create Google Calendar event
    const googleEvent = await createGoogleCalendarEvent({
      customerName,
      customerEmail,
      service,
      appointmentDate,
      garageName,
      garageAddress,
    });

    // 2) Send confirmation email immediately
    const confirmation = await resend.emails.send({
      from: "Jerry Service Garage <appointments@yourdomain.com>",
      to: [customerEmail],
      subject: "Appointment Confirmation",
      html: `
        <p>Hi ${customerName},</p>
        <p>Your appointment has been booked.</p>
        <p><strong>Service:</strong> ${service}</p>
        <p><strong>Time:</strong> ${appointmentDate.toLocaleString()}</p>
        <p><strong>Location:</strong> ${garageAddress}</p>
      `,
    });

    // 3) Build reminder dates
    const { twentyFourHoursBefore, twoHoursBefore } = getReminderDates(appointmentDate);

    // 4) Schedule reminder emails
    let reminder24h = { emailId: null, scheduledFor: twentyFourHoursBefore, status: "skipped" };
    let reminder2h = { emailId: null, scheduledFor: twoHoursBefore, status: "skipped" };

    if (canScheduleWithResend(twentyFourHoursBefore)) {
      const emailId = await scheduleReminderEmail({
        to: customerEmail,
        subject: "Reminder: Your appointment is tomorrow",
        html: `<p>Reminder for your ${service} appointment tomorrow.</p>`,
        scheduledAt: twentyFourHoursBefore,
      });
      reminder24h = { emailId, scheduledFor: twentyFourHoursBefore, status: "scheduled" };
    } else if (twentyFourHoursBefore > new Date()) {
      reminder24h.status = "pendingScheduleLater";
    }

    if (canScheduleWithResend(twoHoursBefore)) {
      const emailId = await scheduleReminderEmail({
        to: customerEmail,
        subject: "Reminder: Your appointment is in 2 hours",
        html: `<p>Reminder for your ${service} appointment in 2 hours.</p>`,
        scheduledAt: twoHoursBefore,
      });
      reminder2h = { emailId, scheduledFor: twoHoursBefore, status: "scheduled" };
    } else if (twoHoursBefore > new Date()) {
      reminder2h.status = "pendingScheduleLater";
    }

    // 5) Save booking
    const docRef = await db.collection("appointments").add({
      customerName,
      customerEmail,
      customerPhone,
      service,
      appointmentAt: admin.firestore.Timestamp.fromDate(appointmentDate),
      timezone: "America/Montreal",
      garageName,
      garageAddress,
      status: "booked",
      googleCalendarEventId: googleEvent.id,
      resend: {
        confirmationEmailId: confirmation.data?.id || null,
        reminder24h: {
          emailId: reminder24h.emailId,
          scheduledFor: admin.firestore.Timestamp.fromDate(reminder24h.scheduledFor),
          status: reminder24h.status,
        },
        reminder2h: {
          emailId: reminder2h.emailId,
          scheduledFor: admin.firestore.Timestamp.fromDate(reminder2h.scheduledFor),
          status: reminder2h.status,
        },
      },
      createdAt: admin.firestore.FieldValue.serverTimestamp(),
      updatedAt: admin.firestore.FieldValue.serverTimestamp(),
    });

    return res.status(200).json({
      success: true,
      appointmentId: docRef.id,
      googleCalendarEventId: googleEvent.id,
    });
  } catch (error) {
    console.error(error);
    return res.status(500).json({ success: false, error: error.message });
  }
}
Step 8: When a booking is rescheduled

This part is extremely important.

When the customer changes the appointment:

Read the appointment document from Firestore
Update the Google Calendar event
Cancel any existing scheduled reminder emails
Recalculate the new reminder times
Schedule the new ones
Update Firestore

Resend supports both update and cancel for scheduled emails.

Safest approach

Instead of trying to be fancy with update, I’d do this:

cancel old reminders
create new reminders

That is easier to reason about.

Example:

async function cancelScheduledEmailIfExists(emailId) {
  if (!emailId) return;

  try {
    await resend.emails.cancel(emailId);
  } catch (err) {
    console.error("Failed to cancel email:", emailId, err.message);
  }
}

Then on reschedule:

await cancelScheduledEmailIfExists(appointment.resend?.reminder24h?.emailId);
await cancelScheduledEmailIfExists(appointment.resend?.reminder2h?.emailId);

// then calculate new reminder times and schedule new ones

Resend says that once an email is cancelled, it cannot be rescheduled, so after cancelling, just create a fresh scheduled email.

Step 9: When a booking is cancelled

When the appointment is cancelled:

mark Firestore appointment as cancelled
cancel any scheduled reminder emails in Resend
update or delete the Google Calendar event

Example logic:

await db.collection("appointments").doc(appointmentId).update({
  status: "cancelled",
  updatedAt: admin.firestore.FieldValue.serverTimestamp(),
});

await cancelScheduledEmailIfExists(appointment.resend?.reminder24h?.emailId);
await cancelScheduledEmailIfExists(appointment.resend?.reminder2h?.emailId);

// update Firestore reminder statuses too
Step 10: Handle bookings more than 30 days out

Because Resend only schedules up to 30 days ahead, you need one extra rule for long-range bookings.

Easy starter version

If garage bookings are usually within 30 days, you can launch without extra automation.

Just:

schedule what is within 30 days
store the rest as pendingScheduleLater
Better full version

Add one small daily job on your existing backend/host that:

looks for appointments with reminders marked pendingScheduleLater
checks whether the reminder time is now within 30 days
schedules the email then
updates Firestore with the new Resend email ID and status

Because you’re not on Blaze, I would run that daily job on:

your existing server if you have one,
or your website host’s scheduled job feature,
or a simple external scheduler that hits a protected endpoint on your server.

The reason I’m suggesting this is specifically because Firebase scheduled functions would push you toward Blaze billing.

Step 11: Add a “repair” endpoint for safety

This is optional, but really smart.

Create a protected server endpoint like:

POST /api/reminders/backfill

What it does:

find appointments where reminder status is pendingScheduleLater
if reminder time is now within 30 days and still in the future, schedule it
save the returned Resend email ID

This protects you if:

a booking was made far in advance
a reminder failed to schedule during the initial booking
you deploy a fix later and want to catch older appointments
